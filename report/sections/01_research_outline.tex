\section{Research Outline}

Hytale is a useful case study for modding because creation is part of its basic identity.
Instead of treating mods only as later fan additions,
Hytale provides tools for asset editing,
model creation,
and Java server plugins.
This makes modding part of the game's development pipeline.

Its modding landscape is split between asset-based and code-based work.
Asset packs can change blocks,
items,
models,
textures,
sounds,
and other data-driven content.
Java server plugins can change commands,
events,
UI,
multiplayer rules,
combat behavior,
and gameplay systems.
For my project,
the most important aspect is this server-side plugin model:
the server becomes the place where rules and interactions can be rewritten,
but only through the systems exposed by the toolkit.

My critical mod,
\textit{Trial of the Goddess},
uses this plugin-based approach.
Instead of making a cosmetic mod,
I created a Java server plugin that changes how a player interacts with a statue,
receives a weapon,
moves through combat,
and completes a ritual.
The goal is to test how far Hytale's official toolkit allows a modder to change
the logic of play,
and where the limits of that pipeline become visible.
